PART 1 — AI 기초

AI를 다루는 3단계 엔지니어링

프롬프트 → 콘텍스트 → 하네스 엔지니어링

초점이 '무엇을 말하나' → '무엇을 보여주나' → '어떻게 실행하나'로 확장됩니다. 뒤 단계가 앞 단계를 대체하는 게 아니라 바깥에서 감싸는 구조입니다.

① 프롬프트 엔지니어링
"무엇을 말하나"
지시·역할·형식·예시 설계
메시지 단위 · 2022~2024
② 콘텍스트 엔지니어링
"무엇을 보여주나"
컨텍스트 창에 담을 정보 설계
세션 단위 · 2025
③ 하네스 엔지니어링
"어떻게 실행하나"
모델을 감싼 실행 시스템 설계
시스템 단위 · 2026~
한 줄 요약: 프롬프트 = 지시, 콘텍스트 = 지식, 하네스 = 실행 체계. 셋은 경쟁이 아니라 겹겹이 쌓이는 층입니다. 콘텍스트·하네스 엔지니어링은 PART 4 컨텍스트 엔지니어링에서 자세히 다룹니다.

할루시네이션 (Hallucination)

AI가 없는 사실을 있는 것처럼 말하는 현상
존재하지 않는 논문 인용 · 없는 법률 설명 · 가상의 통계 수치 제시

AI는 "정답을 아는 것"이 아니라 "그럴듯한 다음 단어를 예측"하는 방식으로 작동하기 때문입니다.

PART 2 — Claude & Cowork

Claude & Anthropic이란?

Anthropic이 개발한 LLM 기반 생성형 AI — 하나의 이름이 서비스이자 브랜드이자 모델 이름으로 쓰입니다.

Anthropic (회사)
전 OpenAI 연구원들이 2021년 창립한 AI 안전(safety) 중심의 연구·제품 회사
세 가지 의미
엔진(LLM) · 서비스(Claude.ai) · 모델 이름(Opus·Sonnet·Haiku) — 맥락에 따라 의미가 달라집니다
같은 엔진, 다른 환경
웹 · Cowork · Code 세 가지로 쓰입니다 — 언제 무엇을 쓰는지는 다음 장에서 →
핵심: "Claude"는 회사(Anthropic)의 모델이자 서비스 이름이라, 대화에서 엔진·앱·모델 중 어느 뜻으로 쓰였는지는 맥락으로 구분하면 됩니다.
PART 2 — 핵심 전환

"대화"가 아니라 "위임"

챗봇은 답을 주고 실행은 내 몫. Cowork는 동료에게 맡기듯 — 결과(목표)만 설명하면 결과물을 만들어 옵니다.

내가 하는 것
  • 결과(목표)를 설명 — "무엇을 원하는지"
  • 필요한 맥락 · 파일 · 도구를 제공
  • 중요한 순간에 승인
Cowork가 하는 것
  • 작업을 계획으로 분해
  • 컨텍스트·파일·도구를 활용해 실행
  • 스스로 검토한 뒤 결과물 산출
마인드셋: 코스 전체를 관통하는 한 가지 — 답을 받아 내가 실행하는 게 아니라, 동료에게 일을 맡기듯 Cowork에 위임합니다.

    PART 3 — 파일 다루기

    워킹 폴더 vs 첨부

    워킹 폴더 (마운트)

    폴더를 마운트하면 그 안의 파일을 읽기·쓰기·편집·삭제할 수 있습니다. 다운로드·데스크탑 폴더에는 기본적으로 접근하지 못합니다.

    주의: 잘못 프롬프트하면 파일을 삭제할 수도 있습니다. 다시 못 구하는 파일은 복사본 폴더로 작업하세요.
    프로젝트 = 워킹 폴더 + 더 편한 UI + 전용 메모리. 작업이 끝나면 결과물도 그 폴더에 저장됩니다.
    첨부 (+ 버튼)

    프롬프트와 함께 주는 파일. 즉시 컨텍스트에 로드되어 바로 읽힙니다.

    언제 무엇을?
    첨부가 맞을 때 — "이 디자인 실수 봐줘" 같은 스크린샷
    워킹 폴더가 맞을 때 — 읽고·수정·생성할 실제 파일(PDF·엑셀·md)

    PART 4 — 기본 개념

    토큰 & 컨텍스트

    토큰 (Token)

    AI가 텍스트를 처리하는 기본 단위. 단어 하나일 수도, 단어보다 작은 조각일 수도 있습니다. (영어 1단어 ≈ 1~2토큰, 한글은 더 잘게 쪼개집니다)

    비용과 직결: 입·출력 토큰 양에 따라 청구됩니다. 명확한 요청으로 한 번에 원하는 결과를 얻는 게 효율적입니다.
    컨텍스트 (Context)

    AI가 현재 대화에서 참고하는 정보의 범위. "아까 그거"라고 해도 이해하는 이유가 바로 컨텍스트입니다.

    더 깊이

    컨텍스트가 쌓이는 '컨텍스트 윈도우'와 이를 설계·관리하는 방법은 바로 이어지는 슬라이드에서 자세히 다룹니다.

    한 줄 정리: 토큰 = 처리·비용의 단위, 컨텍스트 = AI가 참고하는 정보 범위. 둘을 의식하면 더 빠르고 저렴하게 쓸 수 있습니다.
    PART 4 — 사용량 & 한도

    한도는 어떻게 작동하나

    두 종류의 한도
    세션5시간 롤링 윈도우 — 가장 먼저 닿음
    주간한 주 동안의 총 사용량
    Chat·Cowork·Code가 같은 한도를 공유합니다. 정확한 토큰 수는 비공개이며 가용 컴퓨팅에 따라 변동됩니다(설정 → 사용량에서 % 확인). 한도 초과 시 크레딧을 충전해 계속 사용 가능.
    한도를 아끼는 법
  • 긴 대화 피하기 — 매 메시지마다 전체 컨텍스트 재처리
  • 스킬 활용 — 발견(시행착오)을 한 번만, 이후엔 스크립트 실행
  • 단순 질문은 Cowork 말고 Chat에서 — 더 빠르고 저렴
    • PART 4 — 사용량 & 한도

    토큰 절약 — 맥락을 가볍게

    매 턴마다 전체 맥락(파일·지침·이전 대화)이 다시 처리됩니다. 절약의 핵심은 하나 — 맥락을 가볍게 유지하기.

    ① 도구 선택
    • 단순 작업은 Cowork 말고 Chat에서 (더 빠르고 저렴)
    • 일상은 Sonnet, 복잡한 다단계 추론만 Opus · effort도 작업에 맞게
    ② 맥락 최소화
    • 필요 없으면 폴더 0개 · 필요한 파일·도구만 연결
    • 전역 지침·about-me는 짧게 (매 세션 고정 비용)
    • 주제가 바뀌면 새 태스크 — 긴 대화는 비용이 복리
    ③ 재사용
    • 세션 끝에 session-notes.md로 정리 → 다음 세션은 그걸 먼저 읽기
    • 반복 업무는 프로젝트 — 지식이 커지면 자동 RAG로 관련 청크만 검색
    출발점은 측정: 설정 → 사용량에서 어떤 패턴이 토큰을 많이 쓰는지 확인하세요. (코워크 세션 하나가 일반 채팅 여러 개분 사용량을 쓰기도 합니다)

    플러그인 & 라이브 아티팩트

    플러그인 vs 스킬

    스킬 = 특정 작업을 어떻게 할지. 플러그인 = 여러 스킬 + 필요한 커넥터(+에이전트·훅)를 묶어 한 번에 공유·설치.

    데이터 분석을 시키며 "라이브 아티팩트로 만들어줘" → CSV·커넥터 데이터로 인터랙티브 대시보드 생성.

    공유: 팀/엔터프라이즈는 링크로 공유(같은 조직 내), 개인은 웹 퍼블리시 가능. 편집은 만든 대화로 돌아가 피드백을 주면 됩니다.

    PART 6 — 커스텀 연결

    커넥터가 없을 땐? 커스텀 MCP

    연결 흐름
    1
    도구 이름 + "MCP server" 검색예: circle.so MCP server
    2
    개발자 문서에서 서버 URL 확보"Connection URL"을 복사
    3
    Customize → 커넥터 → + → 커스텀 커넥터 추가이름 + 원격 MCP 서버 URL 입력
    4
    Connect 후 로그인(OAuth)
    MCP란?

    Model Context Protocol — 모든 Claude 커넥터가 기반하는 표준 규약. 일단 연결되면 빌트인 커넥터와 기능·권한 설정이 동일합니다.

    새 커넥터 연결 시 습관

    새 도구를 붙였다면 곧장 물어보세요 — "이 커넥터로 가능한 모든 동작과, 우리 워크플로에 맞는 활용 사례를 알려줘." 미처 생각 못 한 기능을 발견하게 됩니다.

    로컬 vs 클라우드

    로컬 (Local) = 내 컴퓨터

    "로컬에서 실행한다" = 내 컴퓨터에서 실행. 파일이 내 하드디스크에 저장됩니다.

    클라우드 (Cloud) = 인터넷 어딘가의 서버

    구글 드라이브·네이버 클라우드가 대표적. 내 파일이 회사 서버 컴퓨터에 저장됩니다.

    Claude에서 이 개념이 중요한 이유
    Claude 웹 버전

    클라우드 방식. Anthropic 서버에서 AI가 돌아가고 결과만 브라우저에 표시 — 내 컴퓨터 파일 접근은 어렵습니다.

    Claude Code / Cowork

    로컬 + 클라우드 조합. AI 두뇌는 클라우드에서 빌리고, 내 컴퓨터 파일을 직접 읽고·쓰고·생성합니다.

    • 팀원과 코드 공유·협업
    Windows는 Git을 별도 설치! winget install Git.Git 또는 git-scm.com. 설치 시 Git Bash · Git Credential Manager가 함께 포함됩니다.
